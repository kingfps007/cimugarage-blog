# 漏洞披露策略

## 支持版本

| 版本 | 支持状态 |
|------|---------|
| 最新版（main 分支） | ✅ 积极维护 |
| 旧版本 | ❌ 不维护，请升级到最新版 |

## 如何报告漏洞

**请勿**在 GitHub Issues 中公开披露安全问题。

请通过以下任一私密渠道联系：

- **邮箱**：2479010668@qq.com（推荐）
- **PGP 公钥**：暂无（如需要可联系站长生成）

### 报告应包含

1. **问题简述**：1-2 句话描述
2. **复现步骤**：详细步骤 + 触发 URL
3. **影响范围**：是全部用户 / 部分用户 / 仅特定条件
4. **建议修复**（如有）
5. **您的联系信息**（可选）

## 响应承诺

| 阶段 | 时间 |
|------|------|
| 确认收到 | 24 小时内 |
| 初步评估 | 3 个工作日内 |
| 修复上线 | 视严重程度（关键 24h，高 7 天，中 30 天，低下次迭代） |
| 公开致谢 | 修复上线后（除非报告者希望匿名） |

## 范围

**在范围内**：

- 评论系统的 XSS / CSRF / 越权
- 浏览数 / 排行榜的注入
- 管理员后台认证绕过
- 隐私政策未声明的数据收集
- 重定向逻辑绕过（绕过白名单）

**不在范围内**（请通过普通渠道反馈）：

- 内容错误、错别字
- 文章内容争议
- 个人观点分歧
- 第三方依赖漏洞（请直接向依赖方报告）

## 安全更新通知

重大安全更新会通过以下渠道公布：

- 仓库 Releases
- 站内公告（如有）
